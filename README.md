## 🎮 게임 소개

**Shape It!**은 제시어를 여러 개의 3D 도형으로 표현하고, 다른 플레이어가 만든 작품을 보고 정답을 맞히는 **최대 8인 온라인 파티 게임**입니다.

플레이어는 무작위 제시어를 받은 뒤 제한 시간 동안 상자, 구, 원기둥 등의 도형을 조합해 자신만의 작품을 제작합니다.

제작 과정을 감상하며 정답을 추측하고 제출합니다. 누구보다 빠르게 정답을 맞히고, 다른 플레이어가 알아볼 수 있는 작품을 만들어 가장 높은 점수를 획득해 보세요!

<br>

## 🔄 게임 루프

```text
행동 (Action)
게임 시작 화면
→ (방장) 로비 생성 및 방 설정
  • 작업 제한 시간 설정
  • 최대 사용 도형 수 설정
→ (플레이어) 로비 참가
→ 방장이 게임 시작
→ 게임 맵 이동 및 플레이어별 제시어·개인 작업공간 배정
  
 • 플레이어별 턴 루프 시작
→ 현재 플레이어는 자신의 작업공간으로 이동
→ 나머지 플레이어는 해당 작업공간을 볼 수 있는 관람 위치로 이동
→ 현재 플레이어에게만 제시어 공개
→ 방장이 설정한 제한 시간 동안 게임 진행
  • 현재 플레이어는 제한된 개수의 도형으로 제시어를 표현
  • 나머지 플레이어는 제작 과정을 실시간으로 보며 정답 입력
→ 정답을 맞힌 플레이어와 제작자에게 점수 지급
→ 모든 추측자가 정답을 맞히면 2초 후 즉시 다음 턴으로 이동
→ 제한 시간이 종료되면 제작 및 정답 입력 종료
→ 다음 플레이어의 작업공간으로 이동하여 동일 과정 반복
 • 플레이어별 턴 루프 종료

→ 마지막 플레이어의 턴 종료
→ 게임 종료 및 최종 점수판 출력
→ 5초 후 로딩 화면 표시
→ 모든 플레이어가 로비로 이동

결과 (Outcome)
각 플레이어의 정답 및 제작 점수가 누적된 최종 점수판 확인
→ 기존 참가자 구성으로 로비 복귀
→ 방 설정 변경 또는 다음 게임 시작 가능
```

## ⚙️ 주요 구현

### 서버 권한 기반 도형 편집

클라이언트가 도형을 직접 생성하지 않고 Server RPC를 통해 서버에 배치를 요청합니다.

서버는 현재 게임 단계, 플레이어의 배치 권한, 최대 도형 개수 등을 검증한 후 도형을 생성하고 모든 클라이언트에 결과를 동기화합니다.

이를 통해 클라이언트마다 도형의 상태가 다르게 보이거나 허용된 개수보다 많은 도형이 생성되는 문제를 방지했습니다.

### 게임 단계 동기화

서버의 `GameMode`가 전체 게임 흐름을 제어하고, `GameState`가 현재 단계, 남은 시간, 라운드와 점수를 클라이언트에 복제합니다.

각 클라이언트의 UI는 복제된 상태와 이벤트를 기반으로 갱신되어 모든 플레이어가 동일한 게임 진행 상황을 확인할 수 있습니다.

### Steam 세션 관리

`GameInstanceSubsystem`을 이용해 세션의 생명주기를 관리했습니다.

- 세션 생성 및 제거
- 온라인 방 검색
- 비밀번호 검증
- 세션 참가 및 퇴장
- 게임 진행 상태 갱신
- 네트워크 실패 감지
- 호스트 연결 종료 처리

Steam을 사용할 수 없는 환경에서는 Online Subsystem 설정을 변경하여 LAN 환경으로 전환할 수 있도록 구성했습니다.

### 정답 및 점수 검증

채팅을 통해 제출된 정답은 서버에서 검증합니다.

- 작품 제작자는 자신의 작품에 정답을 제출할 수 없음
- 한 라운드에서 동일 플레이어의 중복 정답 방지
- 정답 순서에 따라 차등 점수 지급
- 첫 정답 발생 시 제시어 난이도에 따라 제작자 점수 지급
- 추가 정답 발생 시 제작자에게 추가 점수 지급

점수 변경은 서버에서만 처리하여 모든 플레이어에게 동일한 결과가 표시되도록 했습니다.

### 맵 이동과 플레이어 퇴장 구분

Unreal Engine의 `ServerTravel` 과정에서는 기존 플레이어가 일시적으로 로그아웃된 것처럼 처리될 수 있습니다.

이를 실제 플레이어 퇴장과 구분하기 위해 맵 이동 상태를 별도로 관리하고, 이동 과정에서는 퇴장 메시지와 세션 정리 로직이 실행되지 않도록 구성했습니다.

## 🧩 트러블슈팅

### 다른 게임의 Steam 로비가 방 목록에 표시되는 문제

**문제**

Steam 테스트 App ID를 사용하는 다른 프로젝트의 로비까지 Shape It! 방 목록에 함께 표시되는 문제가 있었습니다.

**해결**

세션 생성 시 프로젝트를 구분할 수 있는 식별값을 추가하고, 검색 결과를 표시하기 전에 해당 식별값을 검증하도록 구현했습니다.

**결과**

Shape It!에서 생성된 세션만 방 목록에 표시됩니다.

---

### 레벨 이동을 플레이어 퇴장으로 판단하는 문제

**문제**

로비에서 게임 맵으로 이동하거나 게임 종료 후 로비로 돌아올 때 플레이어가 실제로 나가지 않았는데도 퇴장 안내가 표시되었습니다.

**해결**

`ServerTravel`이 시작됐는지를 나타내는 상태를 별도로 관리하고, 맵 이동 중 발생한 `Logout`과 실제 연결 종료를 구분했습니다.

**결과**

실제 플레이어가 방을 떠났을 때만 퇴장 안내가 표시됩니다.

---

### 호스트 이탈 안내가 사라지는 문제

**문제**

호스트의 연결이 종료되면 메인 메뉴로 이동하는 과정에서 안내 위젯이 생성되었다가 레벨 전환으로 즉시 사라졌습니다.

**해결**

연결 실패 원인을 `GameInstanceSubsystem`에 임시 저장하고, 메인 메뉴의 로딩이 완료된 뒤 해당 정보를 소비하여 안내창을 표시하도록 변경했습니다.

**결과**

레벨이 전환된 이후에도 사용자에게 정확한 연결 종료 원인을 안내할 수 있습니다.

---

### 한글 방 제목 전달 문제

**문제**

세션 설정을 통해 한글 방 제목을 전달할 때 문자열이 정상적으로 검색 결과에 표시되지 않는 문제가 있었습니다.

**해결**

세션에 저장되는 값의 인코딩과 전달 방식을 보완하고, 클라이언트에서 원래 방 제목을 복원하도록 처리했습니다.

**결과**

한글로 설정한 방 제목도 생성 및 검색 과정에서 정상적으로 표시됩니다.

## 🏗 프로젝트 구조

```text
Source/TeamProject
├─ Character/          # 캐릭터 이동과 도형 편집
├─ Component/          # UI 및 기능 컴포넌트
├─ DataAsset/          # 색상 팔레트 데이터
├─ Enums/              # 게임 단계와 공통 데이터 타입
├─ GameInstance/       # Steam 세션 및 연결 상태 관리
├─ Gamemode/           # 로비와 인게임 서버 로직
├─ GameState/          # 게임 진행 상태 복제
├─ Input/              # Enhanced Input 설정 클래스
├─ Object/             # 도형과 작업 공간 오브젝트
├─ PlayerController/   # 입력, UI 및 서버 요청
├─ PlayerState/        # 플레이어 정보와 점수
├─ Structs/            # 공통 구조체
└─ UI/                 # 메뉴, 로비, HUD, 점수판
```

### 주요 클래스

| 클래스 | 역할 |
|---|---|
| `ASIGameMode` | 게임 단계, 라운드, 제시어와 점수 관리 |
| `ASILobbyGameMode` | 로비 참가자 및 방 설정 관리 |
| `ASIGameState` | 단계, 시간, 라운드와 점수 동기화 |
| `ASIPlayerController` | UI 제어, 채팅 및 정답 제출 |
| `ASIPlayerState` | 닉네임, 호스트 여부와 점수 저장 |
| `ASICharacter` | 캐릭터 이동과 도형 배치·편집 |
| `USISessionSubsystem` | Steam 세션 생성, 검색, 참가와 종료 |
| `USIUIManagerComponent` | 메뉴와 위젯 전환 관리 |
